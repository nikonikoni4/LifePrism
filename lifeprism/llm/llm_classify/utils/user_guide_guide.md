# 文档结构说明

本文档采用结构化 JSON 格式，便于 AI 检索和回答用户问题。

## 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 章节级必填 | 唯一标识，便于引用和跳转 |
| `title` | string | ✅ | 标题 |
| `abstract` | string | ✅ | 摘要，用于快速检索匹配 |
| `keywords` | array | 可选 | 关键词列表，提升匹配准确性 |
| `items` | array | 可选 | 子章节标题列表（导航索引）|
| `details` | object | 可选 | 详细属性（位置、功能等静态信息）|
| `content` | array | 可选 | 嵌套的子章节内容 |

## 层级规则

- **章节级**：使用 `abstract` + `items` + `content`（概述 + 索引 + 详情）
- **组件级**：使用 `abstract` + `details` + `content`（概述 + 静态属性 + 可操作元素）
- **操作级**：仅使用 `abstract`（单一功能描述即可）

## 编写注意事项 
1. 关注功能按键的交互，弱化界面显示介绍
2. 无复杂的功能部分，不需要添加content，写明在abstract内即可
3. 不需要写时间筛选等部分（比较简单，不需要说明）
